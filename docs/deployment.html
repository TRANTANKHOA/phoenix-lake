Glossary

Deployment

Phoenix Lake supports multiple deployment modes — from local development to production on AWS.

Deployment modes

Local Docker Compose

Fastest path to a running system. All components in containers on one machine. Best for development and demos.

Dev/Prod Kubernetes

Helm charts for self-managed clusters. Auto-scaling, rolling updates, service discovery. Best for teams running their own K8s.

Production AWS

Managed services: ECS/EKS, RDS, S3. Auto-scaling, backups, IAM. Best for production workloads on AWS.

Docker Compose (local)

Run everything locally with a single command.

Quick start

  1. Clone the repo and run docker compose up.
  2. Postgres starts with the DuckLake catalog schema auto-provisioned.
  3. MinIO (S3-compatible) starts with the phoenix-lake bucket created.
  4. DuckDB service starts and connects to Postgres and MinIO.
  5. Phoenix app starts and connects to all services.

Services

Postgres
Port 5432. User: postgres, password: postgres. Auto-creates phoenix_lake_dev database.
MinIO
API port 9000, console port 9001. User: minioadmin. S3-compatible object storage.
DuckDB
Port 8080. Connects to Postgres (catalog) and MinIO (data). Memory limit: 4GB.
Phoenix
Port 4000. Web UI, API, Oban workers. Connects to all services.

Environment variables

DATABASE_URL=ecto://postgres:postgres@localhost/phoenix_lake_dev
DUCKDB_HOST=localhost
DUCKDB_PORT=8080
S3_ENDPOINT=http://localhost:9000
S3_BUCKET=phoenix-lake
S3_ACCESS_KEY=minioadmin
S3_SECRET_KEY=minioadmin
SECRET_KEY_BASE=dev_only_secret_key_base

Kubernetes (Helm)

Deploy to any Kubernetes cluster with Helm charts.

Chart structure

charts/phoenix-lake/
├── Chart.yaml
├── values.yaml
├── templates/
│   ├── deployment.yaml      # Phoenix app
│   ├── duckdb-service.yaml  # DuckDB compute
│   ├── ingress.yaml         # HTTP routing
│   ├── hpa.yaml             # Auto-scaling
│   └── configmap.yaml       # Environment config

Install

  1. Add the Helm repo: helm repo add phoenix-lake https://charts.phoenix-lake.dev
  2. Install: helm install phoenix-lake phoenix-lake/phoenix-lake --set env=prod
  3. Or from source: helm install phoenix-lake ./charts/phoenix-lake

Key values

postgres
External RDS or in-cluster Postgres. Connection string via DATABASE_URL.
s3
Existing S3 bucket. Credentials via IAM role or secrets.
duckdb.replicas
Number of DuckDB service pods. Default: 2. Auto-scales via HPA.
duckdb.memory
Memory limit per DuckDB pod. Default: 4Gi.
phoenix.replicas
Number of Phoenix pods. Default: 2. Auto-scales via HPA.

AWS (ECS / EKS)

Production deployment on AWS using managed services.

AWS deployment topology

AWS services

Compute
ECS Fargate (serverless) or EKS (self-managed K8s). Phoenix and DuckDB as separate task definitions.
Database
RDS Postgres (Multi-AZ for prod). One instance, two schemas: app metadata + DuckLake catalogs.
Storage
S3 for Parquet data. One bucket, three prefixes: landing/, refining/, reporting/. Lifecycle policies for cleanup.
Networking
ALB for HTTPS termination. Private subnets for ECS tasks. VPC endpoints for S3 and RDS.
IAM
Task roles for Phoenix (RDS + S3) and DuckDB (RDS read-only + S3). No hardcoded credentials.

Terraform (optional)

module "phoenix_lake" {
  source = "./terraform/aws"

  environment     = "prod"
  vpc_id          = module.vpc.vpc_id
  private_subnets = module.vpc.private_subnets
  db_password     = var.db_password

  # Auto-scaling
  phoenix_min_tasks = 2
  phoenix_max_tasks = 10
  duckdb_min_tasks  = 2
  duckdb_max_tasks  = 10
}

Comparison

When to use Docker Compose
  • Local development
  • Demos and prototyping
  • Single-developer workflows
  • CI/CD test environments
When to use Kubernetes
  • Teams already running K8s
  • Multi-tenant deployments
  • Need rolling updates and canary
  • Self-managed infrastructure
When to use AWS ECS/EKS
  • Production workloads on AWS
  • Need managed services (RDS, S3)
  • Auto-scaling without K8s expertise
  • IAM-based security model
Common to all modes
  • Same Docker images
  • Same environment variables
  • Same health checks
  • Same observability (Prometheus/Grafana)

Environment configuration

All modes use the same environment variables. Only the values change.

Required variables

DATABASE_URL
Postgres connection string. ecto://user:pass@host/db
DUCKDB_HOST
DuckDB service hostname. localhost (compose) or service name (K8s).
DUCKDB_PORT
DuckDB service port. Default: 8080.
S3_BUCKET
S3 bucket name. phoenix-lake
S3_REGION
AWS region. us-east-1
SECRET_KEY_BASE
Phoenix secret. Generate with mix phx.gen.secret.

Optional variables

S3_ENDPOINT
Custom S3 endpoint (for MinIO). Not needed for AWS S3.
S3_ACCESS_KEY
S3 credentials. Not needed when using IAM roles.
S3_SECRET_KEY
S3 credentials. Not needed when using IAM roles.
DUCKDB_MEMORY_LIMIT
Per-query memory limit. Default: 4GB.
POOL_SIZE
Postgres connection pool size. Default: 10.

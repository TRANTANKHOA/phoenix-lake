Security invariant: DuckDB trusts the control plane completely. It only receives pre-authorized, table-scoped requests. All auth logic lives in Phoenix — one boundary, enforced once.

4 Schemas

User
email:stringunique, indexed
name:stringfrom IdP or admin
roleenumadmin | editor | viewer
provider:stringgoogle | workos | okta | local
provider_uid:stringIdP's unique user ID
active:booleandefault: true
Token (API Keys)
key_hash:stringbcrypt — never plaintext
key_prefix:stringfirst 8 chars for lookup
name:string"CI pipeline", "dbt runner"
scopes:array["read", "write"]
expires_at:utc_datetimeoptional expiration

Grant

database:string"landing", "refining", "reporting", or *
table:stringtable name or * for all tables
permissionenumread | write | admin
Wildcard examples: database: "*", table: "*" → full access. database: "landing", table: "orders" → only orders in landing.

5 Roles & IdP Mapping

🔑

admin

  • Full access to all databases and tables
  • Manage users, tokens, grants
  • Map from IdP group: data-admins
✏️

editor

  • Read + write data across databases
  • Create/drop tables, ingest data
  • Map from IdP group: data-engineers
👁️

viewer

  • Read-only access via grants
  • Run queries, browse datasets
  • Map from IdP group: data-analysts
⚙️

default_role

  • Applied to unmapped IdP users
  • Configurable per provider
  • Usually viewer
Multiple groups? Highest role wins: admin > editor > viewer. Mapping is configurable per provider in config/runtime.exs.

6 Supported Providers

Google

OAuth2 via Assent.Strategy.Google. Groups from Admin SDK or directory sync.

WorkOS

SSO via Assent.Strategy.OAuth2. SAML + OIDC connections. Groups from WorkOS directory.

Okta

OIDC via Assent.Strategy.OIDC. Groups from Okta app assignment or directory.

Custom

Any OAuth2/OIDC provider via Assent base strategies. Implement normalize/2.

8 Design Decisions

# Issue Decision Why
D1 hash_pwd_salt generates new salt each call Prefix lookup → Bcrypt.verify_pass Prefix narrows to 1 row; verify is constant-time
D2 expires_at not checked Check after bcrypt; reject if past Tokens revocable by time
D3 WorkOS OAuth2 vs OIDC Keep Assent.Strategy.OAuth2 Simpler, no OIDC discovery needed
D4 Google doesn't return groups in userinfo Admin SDK at login, or Oban sync Real-time vs eventual consistency tradeoff
D5 No token usage visibility Add last_used_at, update on validation Audit trail, cleanup, security monitoring
D6 Concurrent OAuth logins race on insert on_conflict: :nothing + unique index Race-safe, no duplicates
